# Code Of Conduct

## Our Standard

RedDial contributors and maintainers should make participation respectful, technically rigorous, and safe. Healthy disagreement about APIs, evaluation methodology, and model behavior is welcome; harassment, intimidation, discrimination, sexualized conduct, personal attacks, and publishing another person's private information are not.

Examples of constructive participation include:

- criticizing code or claims with evidence rather than attacking people;
- stating assumptions and uncertainty clearly;
- respecting requests to stop disruptive behavior;
- protecting credentials, private transcripts, and customer data; and
- accepting maintainer decisions while remaining free to propose alternatives.

## Enforcement

Maintainers may edit or remove comments, issues, pull requests, code, or other contributions that violate this standard. Repeated or severe violations may result in a temporary or permanent ban from project spaces.

Report conduct concerns privately to the maintainer using the contact information on [@chokonaira's GitHub profile](https://github.com/chokonaira). Do not use a public issue. Reports will be reviewed promptly and handled as confidentially as practical.

This code of conduct applies in repository spaces and when someone is publicly representing the RedDial project.
